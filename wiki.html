üèóÔ∏è Content Types & Schemas

The wiki consists of 10 main content types, each with specific schemas defined in src/content/config.ts

ü™ê Planets

Primary content type. Each planet is a detailed world with lore, resources, and connections.

Schema Fields: name, description, type, biome, atmosphere_type, habitability, primary_resources, gravity_source, nickname, scientific_interest, year_length_days, day_length_hours

Directory: src/content/planets/

Naming: 001_planetname.md (3-digit ID + name)

üë• Species

Alien civilizations. Describes intelligent life forms found on planets.

Schema Fields: species_id, name, scientific_name, origin, description, classification, habitat, photos

Directory: src/content/species/

üíé Resources

Materials and substances. Unique resources found on planets with specific properties.

Schema Fields: resource_id, name, type, origin, description, rarity, uses, photos

Directory: src/content/resources/

üè¢ Organizations

Galactic institutions. Research groups, corporations, and other organized entities.

Schema Fields: organization_id, name, type, location, description, purpose, membership, influence, photos

Directory: src/content/organizations/

üî¨ Technology

Advanced technologies. Scientific innovations and alien tech.

Schema Fields: technology_id, name, origin, description, type, complexity, photos

Directory: src/content/technology/

ü§ù Relationships

Connections between entities. How different elements interact.

Schema Fields: relationship_id, name, type, participants, description, photos

Directory: src/content/relationships/

üìç Places

Specific locations. Notable places within planets or space.

Schema Fields: place_id, name, location, description, type, significance, photos

Directory: src/content/places/

üèîÔ∏è Features

Planetary features. Geological or environmental characteristics.

Schema Fields: feature_id, name, planet, type, description, photos

Directory: src/content/features/

üí≠ Concepts

üìù Content Creation Guidelines for AI Agents

‚ö†Ô∏è Critical Requirements

  • Always follow the exact YAML frontmatter structure for each content type
  • Use sequential numbering for planets (001, 002, etc.)
  • Reference existing content when creating connections
  • Maintain consistent tone: mysterious yet scientific
  • Include human perspective in all major content

üéØ Content Creation Workflow

  1. Research Existing Content: Review related planets, species, organizations
  2. Identify Connections: Determine how new content relates to existing lore
  3. Create Primary Content: Write the main content file with proper schema
  4. Create Supporting Content: Add related resources, relationships, etc.
  5. Update Cross-References: Add links in existing content where appropriate

üìä Quality Checklist

  • ‚úÖ Frontmatter follows exact schema format
  • ‚úÖ Content includes visual descriptions for rendering
  • ‚úÖ Mysterious elements balance with scientific plausibility
  • ‚úÖ Human perspective included (expeditions, research, reactions)
  • ‚úÖ Connections to at least 2-3 existing entities
  • ‚úÖ Consistent terminology with existing content
  • ‚úÖ Unresolved mysteries for future development

üö´ Common Pitfalls to Avoid

  • Don't make planets too Earth-like
  • Don't resolve all mysteries completely
  • Don't ignore cross-references to existing content
  • Don't use inconsistent naming conventions
  • Don't create content in isolation - always connect to existing lore
